# Plan: Simple Well-Calibrated Simulation Study (WCSS)

## Goal

Validate Delphy's MCMC inference by running a well-calibrated simulation study.
Simulate N datasets from known priors, run Delphy on each, then check that
the inferred posteriors recover the true generating values at the expected rates
(~95% coverage for 95% HPD intervals, uniform rank distributions).

## Configuration

This simple study introduces as few variables as possible to permit setting up
the WCSS infrastructure.  Later studies relax many of these initial restrictions.

- **Tips:** 200, named `TIP_000` through `TIP_199`, dates uniformly distributed over 2025
- **N (number of simulation replicates):** 200
- **Fixed parameters (same across all replicates):**
  - Mutation rate mu = 1e-3 subst/site/year
  - Effective population size N_e = 1.0 year (constant population model)
  - No site rate heterogeneity
  - Genome size L = 30,000 sites
- **Sampled parameters (drawn fresh per replicate):**
  - Stationary frequencies (pi_A, pi_C, pi_G, pi_T) ~ Dirichlet(1,1,1,1)
  - HKY kappa ~ LogNormal(meanlog=1.0, sdlog=1.25)
    (i.e., log(kappa) ~ Normal(mean=1.0, std=1.25); this matches Delphy's hardcoded prior)

## Deliverables

Files in `wcss/02_simple/`:

1. **`01_generate.py`** ‚Äî Python script that generates all simulation inputs and the Makefile
2. **`02_run.py`** ‚Äî Convenience script that invokes `make -jN`; accepts a `--jobs` option (default: half the available CPUs). The `Makefile` is an implementation detail, generated by `01_generate.py`.
3. **`03_analyze.py`** ‚Äî Python script that runs loganalyser, checks ESS, computes coverage/ranks, and writes all results to TSV files under `analyses/`
4. **`04_plot.py`** ‚Äî Python script that reads TSV files from `analyses/` and produces all plots (no computation)

Plus directories created at runtime:
- **`sims/`** ‚Äî Makefile, tips file, and per-replicate simulation directories
  - `Makefile` ‚Äî drives Delphy runs (generated by `01_generate.py`)
  - `tips.txt` ‚Äî shared tip file (generated by `01_generate.py`)
  - `sim_NNN/` ‚Äî per-replicate simulation inputs, Delphy outputs, and completion markers
- **`analyses/`** ‚Äî analysis outputs (generated by `03_analyze.py`)
  - `coverage_summary.txt` ‚Äî coverage results in TSV format
  - `ess_check.tsv` ‚Äî per-observable ESS check results
  - `loganalyser_output.tsv` ‚Äî raw loganalyser output
  - `ranks.tsv` ‚Äî normalized ranks per parameter per replicate
  - `true_params.tsv` ‚Äî true parameter values per replicate
- **`plots/`** ‚Äî all generated figures (generated by `04_plot.py`)

## Part 1: `01_generate.py` ‚Äî Generate

### Step 1: Generate the tips file

- Write `tips.txt` in `wcss/02_simple/sims/` in Sapling's tip file format:
  `Name|YYYY-MM-DD`, one per line.
- 200 tips named `TIP_000` through `TIP_199`.
- Dates drawn uniformly over 2025-01-01 to 2025-12-31.
- Use a fixed seed for reproducibility (the tips represent the "sampling design",
  not a random variable being inferred).

### Step 2: For each replicate i = 0..N-1

1. **Sample parameters from the prior:**
   - Draw (pi_A, pi_C, pi_G, pi_T) ~ Dirichlet(1,1,1,1) using `numpy.random`.
   - Draw kappa ~ LogNormal(meanlog=1.0, sdlog=1.25).
   - Use a per-replicate seed derived from a master seed (e.g., `master_seed + i`).

2. **Run Sapling to simulate data:**
   - Create directory `sims/sim_{i:03d}/`.
   - Invoke Sapling via `subprocess`:
     ```
     sapling \
       --tip-file sims/tips.txt \
       --t0 2026-01-01 \
       --const-pop-n0 1.0 \
       --mu 1e-3 \
       --hky-kappa {kappa} \
       --hky-pi-A {pi_A} --hky-pi-C {pi_C} --hky-pi-G {pi_G} --hky-pi-T {pi_T} \
       --num-sites 30000 \
       --seed {replicate_seed} \
       --out-maple sims/sim_{i:03d}/sim.maple \
       --out-info sims/sim_{i:03d}/sim_info.json \
       --out-newick sims/sim_{i:03d}/sim.nwk \
       --out-nexus sims/sim_{i:03d}/sim.nexus \
       --out-fasta sims/sim_{i:03d}/sim.fasta
     ```
   - `t0` is set to 2026-01-01 so that when we later switch to exponential
     population curves, `--exp-pop-n0` has the natural interpretation of the
     effective population size at the *end* of the sampling period.
   - The `sim_info.json` records all generating parameters (ground truth).

### Step 3: Generate the Makefile

Generate a `Makefile` with:

- The Makefile lives in `sims/`, so all paths are relative to that
  directory.
- A `SIM_DIRS` variable using `$(wildcard sim_[0-9]*)` to discover
  all simulation directories, and a default target `all` depending on
  `$(addsuffix /.done,$(SIM_DIRS))`.  This makes the Makefile contents
  completely independent of N.
- A single pattern rule using `%` (which matches the replicate tag, e.g.,
  `000`).  Filenames within each `sim_NNN/` directory do not include the
  replicate number, so the prerequisite only needs one `%`.  `$*` provides
  the stem in the recipe:
  ```makefile
  SIM_DIRS := $(wildcard sim_[0-9]*)

  all: $(addsuffix /.done,$(SIM_DIRS))

  sim_%/.done: sim_%/sim.maple
  	../../../delphy \
  	  --v0-in-maple $< \
  	  --v0-steps 500000000 \
  	  --v0-out-log-file sim_$*/delphy.log \
  	  --v0-log-every 500000 \
  	  --v0-out-trees-file sim_$*/delphy.trees \
  	  --v0-tree-every 500000 \
  	  --v0-out-delphy-file sim_$*/delphy.dphy \
  	  --v0-delphy-snapshot-every 500000 \
  	  --v0-init-mutation-rate 1e-3 \
  	  --v0-fix-mutation-rate \
  	  --v0-init-final-pop-size 1.0 \
  	  --v0-fix-final-pop-size \
  	  --v0-init-pop-growth-rate 0.0 \
  	  --v0-fix-pop-growth-rate \
  	&& touch $@
  ```
- The `&& touch $@` pattern ensures the `.done` marker is only created if Delphy
  exits successfully.  `make -jK` then provides tunable parallelism.
- `01_generate.py` accepts a `--steps` option (default 500,000,000) to control
  the number of MCMC steps per replicate.  This value is written into the
  Makefile.  If ESS is too low, regenerate the Makefile with a higher
  `--steps` value and rerun.
- Delphy run parameters:
  - `--v0-steps`: controlled by `--steps` (default ~2,500,000 per tip =
    500,000,000 for 200 tips).
  - `--v0-log-every` and `--v0-tree-every`: steps / 1000, yielding
    ~1000 log entries and ~1000 trees for more accurate ESS measurement.
  - Also output `.dphy` files with the same snapshot interval.
  - Fix mutation rate, population size, and growth rate to their true values
    so we're only validating kappa, stationary frequencies, and tree height.
- A `clean` target that removes all `sim_*/delphy.*` and `sim_*/.done` files.

## Part 3: `03_analyze.py` ‚Äî Analyze

This script performs all computation and writes results to TSV files under
`analyses/`.  It produces no plots.

### External tool: BEAST 2's `loganalyser`

Available at `../../loganalyser277`.  When invoked as:
```
../../loganalyser277 -oneline -burnin 30 file1.log file2.log ...
```
it produces TSV output with a single header line followed by one row per
input file (in command-line order).  Each row contains, for every observable
in the log file: `{name}.mean`, `{name}.95%HPDlo`, `{name}.95%HPDup`,
`{name}.ESS`, `{name}.median`, etc.  The `-burnin 30` flag enforces 30%
burn-in (the default is 10%).

The relevant columns for our parameters are:
- `kappa.mean`, `kappa.95%HPDlo`, `kappa.95%HPDup`, `kappa.ESS`
- `frequencies1.mean` / `.95%HPDlo` / `.95%HPDup` / `.ESS` (= pi_A)
- `frequencies2.*` (= pi_C), `frequencies3.*` (= pi_G), `frequencies4.*` (= pi_T)
- `rootHeight.mean` / `.95%HPDlo` / `.95%HPDup` / `.ESS`

### Step 1: Run loganalyser and ESS check

Call `../../loganalyser277 -oneline -burnin 30 sims/sim_000/delphy.log sims/sim_001/delphy.log ...`
via `subprocess`.  Save the raw TSV output to `analyses/loganalyser_output.tsv`
so that a human can inspect it or run further analyses later.
Then parse it into a pandas DataFrame.  Each row corresponds to one replicate,
indexed in the same order as the command-line arguments.

Immediately after parsing, check ESS values.  For each observable reported
by loganalyser, compute the number of replicates where `{name}.ESS` < 200.
Save a TSV file (`analyses/ess_check.tsv`) listing *all* observables,
with columns: observable name, number of low-ESS runs, total runs, fraction,
mean ESS, standard deviation of ESS, and whether the observable is in the
ignore list.  Example:

```
Observable	Low_ESS_runs	N	Fraction	Mean_ESS	Std_ESS	Ignored
kappa	0	200	0.000	512.3	98.7	no
frequencies1	18	200	0.090	245.1	112.4	no
meanRate	200	200	1.000	3.2	1.1	yes
...
```

Also print a human-readable summary to stdout showing only non-ignored
observables with at least one low-ESS run.

Some observables are expected to have low ESS and should be ignored
(e.g., fixed parameters like `meanRate`).  Maintain an `ESS_IGNORE` set
in the configuration section listing observable names to skip.  In this
study, `ESS_IGNORE = {"meanRate"}` (mutation rate is fixed).

If any non-ignored observable has low-ESS runs in more than 5% of
replicates, the script exits with an error message directing the user to
increase `--steps` in `01_generate.py` and rerun.  If the fraction is
below 5%, the script prints a warning but continues.  The flag
`--ignore-low-ess` overrides the error and allows the analysis to
continue regardless.

### Step 2: Read true parameters

For each replicate i = 0..N-1, read from `sims/sim_{i:03d}/sim_info.json`:
- `kappa` from `info['subst_model']['kappa']`
- `pi_A..T` from `info['subst_model']['pi']` (list of 4 floats)
- `rootHeight` from `info['tree_stats']['tree_height']` (time in years
  from the root to the most recent tip; directly comparable to Delphy's
  `rootHeight` log column).

Save as `analyses/true_params.tsv` with columns:
`replicate`, `kappa`, `pi_A`, `pi_C`, `pi_G`, `pi_T`, `rootHeight`.

### Step 3: Coverage analysis

For each parameter:
- Read the 95% HPD interval from the loganalyser output (`{name}.95%HPDlo`,
  `{name}.95%HPDup`).
- Check if the true value falls within `[hpd_lo, hpd_hi]`.
- Tally across all N replicates.
- Report coverage = (number covered) / N.  Expected: ~0.95.

Print a coverage summary table to stdout (human-readable) and save it as a TSV file
(`analyses/coverage_summary.txt`).  Include columns for the expected coverage
(0.95) and the 2.5%/97.5% quantiles of the Binomial(N, 0.95) distribution
(divided by N), written to 3 decimal places.  Example output:

```
Parameter	Coverage	N	Expected	Binom_2.5%	Binom_97.5%
kappa	0.96	200	0.950	0.920	0.980
pi_A	0.95	200	0.950	0.920	0.980
pi_C	0.96	200	0.950	0.920	0.980
pi_G	0.96	200	0.950	0.920	0.980
pi_T	0.98	200	0.950	0.920	0.980
rootHeight	0.94	200	0.950	0.920	0.980
```

### Step 4: Rank Uniformity Validation (RUV)

loganalyser does not compute ranks, so for each replicate we still need
the raw posterior samples.  Read each Delphy log file with
`pd.read_table(filename, comment='#')`, apply the same burnin fraction
as used for loganalyser (30%), and compute:
- `rank_i = sum(posterior_samples < true_value)`
- Normalize to `rank_i / L` where L = number of post-burnin samples,
  giving the empirical posterior CDF evaluated at the true value.

Save as `analyses/ranks.tsv` with columns:
`replicate`, `kappa`, `pi_A`, `pi_C`, `pi_G`, `pi_T`, `rootHeight`.

## Part 4: `04_plot.py` ‚Äî Plot

This script reads TSV files from `analyses/` and produces all plots.
It performs no computation.

### Inputs (all under `analyses/`)

- `loganalyser_output.tsv` ‚Äî posterior means and HPD intervals
- `true_params.tsv` ‚Äî true parameter values per replicate
- `ranks.tsv` ‚Äî normalized ranks per replicate
- `coverage_summary.txt` ‚Äî coverage per parameter

### Outputs (all under `plots/`)

Generate a single multi-panel figure (`wcss_summary.pdf`) with one row per
parameter and three columns:

1. **Rank histogram:** ~N/10 bins on [0, 1], with expected-count line.

2. **ECDF:** empirical CDF of normalized ranks, diagonal reference line,
   and 95% confidence band (x ¬± 1.96 * sqrt(x*(1-x)/N)).
3. **True vs inferred scatter:** x = true value, y = posterior mean (from
   loganalyser `{name}.mean`), with vertical HPD bars (from loganalyser
   `{name}.95%HPDlo` / `.95%HPDup`).  Bars colored light blue if they
   cover the true value, red otherwise.  Zoomed inset for kappa (0‚Äì5).

Also generate individual per-parameter plots (rank histograms, ECDFs,
scatterplots) in `plots/`.

## Execution workflow

```bash
cd wcss/02_simple

# 1. Generate all simulation inputs + Makefile
./01_generate.py --n 200

# 2. Run all Delphy jobs
./02_run.py                # uses half the CPUs by default
./02_run.py --jobs 20      # or specify parallelism explicitly

# 3. Analyze results
./03_analyze.py --n 200

# 4. Plot results
./04_plot.py

# If step 3 fails due to low ESS, increase the number of steps and rerun:
./01_generate.py --n 200 --steps 1000000000
make -C sims clean && ./02_run.py
./03_analyze.py --n 200
./04_plot.py
```
