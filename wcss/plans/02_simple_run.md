# Plan: Simple Well-Calibrated Simulation Study (WCSS)

## Goal

Validate Delphy's MCMC inference by running a well-calibrated simulation study.
Simulate N datasets from known priors, run Delphy on each, then check that
the inferred posteriors recover the true generating values at the expected rates
(~95% coverage for 95% HPD intervals, uniform rank distributions).

## Configuration

This simple study introduces as few variables as possible to permit setting up
the WCSS infrastructure.  Later studies relax many of these initial restrictions.

- **Tips:** 200, named `TIP_000` through `TIP_199`, dates uniformly distributed over 2025
- **N (number of simulation replicates):** 10 initially, scaling to 100+
- **Fixed parameters (same across all replicates):**
  - Mutation rate mu = 1e-3 subst/site/year
  - Effective population size N_e = 1.0 year (constant population model)
  - No site rate heterogeneity
  - Genome size L = 30,000 sites
- **Sampled parameters (drawn fresh per replicate):**
  - Stationary frequencies (pi_A, pi_C, pi_G, pi_T) ~ Dirichlet(1,1,1,1)
  - HKY kappa ~ LogNormal(meanlog=1.0, sdlog=1.25)
    (i.e., log(kappa) ~ Normal(mean=1.0, std=1.25); this matches Delphy's hardcoded prior)

## Deliverables

Three files in `wcss/02_simple/`:

1. **`00_generate.py`** — Python script that generates all simulation inputs and the Makefile
2. **`Makefile`** — generated by `00_generate.py`, drives all Delphy runs
3. **`10_analyze.py`** — Python script that collects results and produces coverage/rank plots
   (stub for now; will be fleshed out later)
4. **`tips.txt`** — shared tip file (generated by `00_generate.py`)

Plus directories created at runtime:
- `sim_NNN/` — per-replicate simulation inputs, Delphy outputs, and completion markers

## Part 1: `00_generate.py`

### Step 1: Generate the tips file

- Write `tips.txt` directly in `wcss/02_simple/` in Sapling's tip file format:
  `Name|YYYY-MM-DD`, one per line.
- 200 tips named `TIP_000` through `TIP_199`.
- Dates drawn uniformly over 2025-01-01 to 2025-12-31.
- Use a fixed seed for reproducibility (the tips represent the "sampling design",
  not a random variable being inferred).

### Step 2: For each replicate i = 0..N-1

1. **Sample parameters from the prior:**
   - Draw (pi_A, pi_C, pi_G, pi_T) ~ Dirichlet(1,1,1,1) using `numpy.random`.
   - Draw kappa ~ LogNormal(meanlog=1.0, sdlog=1.25).
   - Use a per-replicate seed derived from a master seed (e.g., `master_seed + i`).

2. **Run Sapling to simulate data:**
   - Create directory `sim_{i:03d}/`.
   - Invoke Sapling via `subprocess`:
     ```
     sapling \
       --tip-file tips.txt \
       --t0 2026-01-01 \
       --const-pop-n0 1.0 \
       --mu 1e-3 \
       --hky-kappa {kappa} \
       --hky-pi-A {pi_A} --hky-pi-C {pi_C} --hky-pi-G {pi_G} --hky-pi-T {pi_T} \
       --num-sites 30000 \
       --seed {replicate_seed} \
       --out-maple sim_{i:03d}/sim_{i:03d}.maple \
       --out-info sim_{i:03d}/sim_{i:03d}_info.json \
       --out-newick sim_{i:03d}/sim_{i:03d}.nwk \
       --out-nexus sim_{i:03d}/sim_{i:03d}.nexus \
       --out-fasta sim_{i:03d}/sim_{i:03d}.fasta
     ```
   - `t0` is set to 2026-01-01 so that when we later switch to exponential
     population curves, `--exp-pop-n0` has the natural interpretation of the
     effective population size at the *end* of the sampling period.
   - The `sim_{i:03d}_info.json` records all generating parameters (ground truth).
   - Filenames include the simulation number to avoid confusion.

### Step 3: Generate the Makefile

Generate a `Makefile` with:

- A variable `N` for the number of replicates.
- A default target `all` depending on all completion markers `sim_NNN/.done`.
- A pattern rule for each replicate:
  ```makefile
  sim_%/.done: sim_%/sim_%.maple
  	../../delphy \
  	  --v0-in-maple $< \
  	  --v0-steps 20000000 \
  	  --v0-out-log-file sim_$*/delphy_$*.log \
  	  --v0-log-every 20000 \
  	  --v0-out-trees-file sim_$*/delphy_$*.trees \
  	  --v0-tree-every 20000 \
  	  --v0-out-delphy-file sim_$*/delphy_$*.dphy \
  	  --v0-delphy-snapshot-every 20000 \
  	  --v0-init-mutation-rate 1e-3 \
  	  --v0-fix-mutation-rate \
  	  --v0-init-final-pop-size 1.0 \
  	  --v0-fix-final-pop-size \
  	  --v0-init-pop-growth-rate 0.0 \
  	  --v0-fix-pop-growth-rate \
  	&& touch $@
  ```
- The `&& touch $@` pattern ensures the `.done` marker is only created if Delphy
  exits successfully.  `make -jK` then provides tunable parallelism.
- Delphy run parameters:
  - `--v0-steps`: ~100,000 per tip = 20,000,000 for 200 tips.  Start here;
    increase if ESS is too low.
  - `--v0-log-every` and `--v0-tree-every`: steps / 1000 = 20,000, yielding
    1000 log entries and 1000 trees for more accurate ESS measurement.
  - Also output `.dphy` files with the same snapshot interval.
  - Fix mutation rate, population size, and growth rate to their true values
    so we're only validating kappa, pi_a, and tree inference.
- A `clean` target that removes all `sim_*/delphy*` and `sim_*/.done` files.

## Part 2: `10_analyze.py`

This will be fleshed out later.  The outline below captures the intended approach.

### Step 1: Collect results

For each replicate i = 0..N-1:
1. Read true parameters from `sim_{i:03d}/sim_{i:03d}_info.json`
   (kappa, pi_A..T, tMRCA).  Build the table of true parameter values in memory.
2. Read the Delphy log file `sim_{i:03d}/delphy_{i:03d}.log`
   (tab-separated, BEAST-format).
   - Extract posterior samples for: `hky_kappa`, `pi_A`, `pi_C`, `pi_G`, `pi_T`.
   - Apply a burnin (default 30%, configurable).
3. Read the Delphy trees file `sim_{i:03d}/delphy_{i:03d}.trees` to extract
   root times (tMRCA) from each posterior tree sample (parse the Nexus-format
   trees for root node height/date).

### Step 2: Coverage analysis

For each parameter:
- Compute the 95% HPD interval from the posterior samples.
- Check if the true value falls within the interval.
- Report overall coverage fraction across all N replicates.
- Expected: ~95% if the model is correct.

Generate a coverage summary table printed to stdout.

### Step 3: Rank Uniformity Validation (RUV)

For each parameter:
- Compute the rank of the true value among the L posterior samples
  (i.e., how many posterior samples are less than the true value).
- Collect ranks across all N replicates.

Generate plots:
- **Rank histogram** for each parameter (should be approximately flat).
- **ECDF plot** for each parameter (should follow the diagonal).
- Save plots to `wcss/02_simple/plots/`.

### Step 4: Scatterplots

For each parameter:
- Plot true value (x-axis) vs posterior mean (y-axis) across all N replicates.
- Add the identity line for reference.
- Save to `wcss/02_simple/plots/`.

## Execution workflow

```bash
cd wcss/02_simple

# 1. Generate all simulation inputs + Makefile
python 00_generate.py --n 10

# 2. Run all Delphy jobs (e.g., 4 in parallel)
make -j4

# 3. Analyze results
python 10_analyze.py
```
